# ⚡ POWER+ MODE

> **POWER+** kombiniert die stärksten architektonischen und methodischen Skills in einem einzigen, rigorosen Workflow.
> Dieser Modus vereint:
> 1. **Superpowers-DkZ:** Systematische Agenten-Methodik (Brainstorming → Plan → Execution).
> 2. **OpenSpec:** Spec-Driven Development (PRDs, Architektur-Pläne vor Code).
> 3. **Grill-With-Docs:** Gnadenloses Hinterfragen der Architektur gegen das existierende Domain Model (Glossary, ADRs, CONTEXT.md).

---

## 🛠 Ausführungs-Workflow (Pflicht-Reihenfolge)

Wenn der User `/power+` aufruft, folge EXAKT diesem Ablauf:

### Phase 1: The Grill Session (`/grill-with-docs`)
Befrage den User tiefgreifend zu seinem Vorhaben.
- Nutze den `grill-with-docs` Skill.
- Prüfe bestehende `CONTEXT.md` und `ADR`s im Projekt.
- Kläre verschwommene Begrifflichkeiten und Architektur-Entscheidungen, bis ein gemeinsames Verständnis herrscht.

### Phase 2: OpenSpec Definition (`/openspec`)
Nachdem die Grill-Session erfolgreich abgeschlossen ist, schreibe die Spezifikation.
- Generiere oder aktualisiere die `.spec.md` (Product Requirements Document).
- Definiere strikte Akzeptanzkriterien, bevor eine Zeile Code geschrieben wird.

### Phase 3: Superpowers Execution (`/superpowers`)
Führe den Task basierend auf der Spezifikation aus.
- Nutze Bite-sized Tasks (TDD).
- 4-Phasen Debugging bei Fehlern (Root Cause → Pattern → Hypothesis → Fix).
- Verifizierung nach JEDEM Schritt.

Wenn dieser Skill aufgerufen wird, überspringst du NIEMALS die Grill-Session. Bevor du "Ja, mache ich" sagst und anfängst zu coden, **MUSST** du den User grillen.
